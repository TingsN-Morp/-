本仓库提供一个用于重庆医科大学教务系统“教师评教”页面的自动化评教脚本（半成品）。脚本通过浏览器开发者控制台执行，旨在减少重复性操作，帮助学生在复习备考期间节约时间。

脚本运行环境：已于 2026 年 1 月在 Edge 浏览器中测试通过，但不保证兼容所有浏览器或教务系统版本更新。

功能特性：
自动确认弹窗：覆盖 window.confirm 方法，自动确认所有弹窗，避免中断。
智能评教选择：将所有评教项默认勾选为“好”（value="2"），为避免全“好”引发的异常，脚本会取消第一个“好”选项，并任意选择一个未被勾选的“较好”（value="1"）作为均衡项。
自动提交：定位提交按钮并触发点击。
自动返回：利用 sessionStorage 标记提交状态，在页面刷新或跳转后自动点击关闭按钮，返回评教主页面。

代码逻辑说明：
弹窗拦截：window.confirm 被重写为始终返回 true，确保后续操作不被弹窗阻塞。
状态标记：使用 sessionStorage.setItem('needClickClose', 'true') 记录脚本已执行，待页面刷新后触发关闭按钮点击。
评教项选择：
document.querySelectorAll('[value="2"]') 获取所有“好”选项并全部选中。
取消第一个“好”选项（firstValue2.checked = false）。
查找任意一个未被选中的“较好”选项（[value="1"]:not([checked])）并选中。
提交与返回：点击提交按钮后，延迟 1.5 秒再次尝试点击关闭按钮（覆盖未刷新场景）；若页面刷新，则通过初始的 sessionStorage 判断执行关闭。

注意事项：
脚本中“均衡项”的选择逻辑为任意位置的“较好”，并非严格随机选择某评分项目，可能不符合某些用户预期。建议根据实际页面结构调整选择策略。
提交后若未自动返回，可手动点击关闭按钮或刷新页面观察脚本是否触发自动关闭。
若教务系统更新导致页面元素选择器（如 [title="关闭"]、#submitButton 等）失效，脚本将无法正常工作，需自行调整。
